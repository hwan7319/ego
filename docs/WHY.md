# WHY

## 목적
모든 시스템은 구현 전에 "왜 존재해야 하는가"를 먼저 증명한다.

EGO에서 기능은 많을수록 좋은 것이 아니라, 이유가 선명할수록 강하다.

## 의사결정 순서
1. 왜 만드는가
2. 무엇을 만드는가
3. 어떻게 만드는가

위 순서가 뒤집히면 해당 안건은 보류한다.

## 통과 기준 (WHY Gate)
각 안건은 아래 질문에 모두 답해야 한다.

### 1) 존재 이유
- 이 시스템이 해결하려는 플레이어 문제는 무엇인가?
- 이 시스템이 없으면 어떤 경험이 사라지는가?

### 2) 철학 정합성
- 이 안건이 EGO 헌법과 어떻게 연결되는가?
- "모든 영웅에게는 필요한 순간이 있다"를 강화하는가, 약화하는가?
- 플레이어 자유(EGO 성장)를 넓히는가, 좁히는가?

### 3) 검증 가능성
- "유저가 좋아할 것 같다"가 아니라 무엇으로 검증할 것인가?
- 검증 수단은 무엇인가? (유저 인터뷰/플레이 테스트/데이터)
- 실패 신호는 무엇이며, 언제 중단할 것인가?

### 4) 운영 가능성
- 라이브 운영 시 어떤 부작용이 예상되는가?
- 메타 고착 시 너프 외 대안(새 놀이터/새 규칙)을 준비했는가?

## 삭제 규칙
아래 중 하나라도 충족하면 삭제 또는 재설계한다.
- 존재 이유를 설명하지 못한다.
- 헌법 핵심 4원칙과 충돌한다.
- 검증 계획이 없다.
- 운영 시나리오가 없다.

## 금지어
- "다른 게임도 이렇게 하니까."
- "원래 모바일 게임은..."
- "유저들은 이런 거 안 좋아해." (근거 없는 단정)
- "그냥." (이유 없는 결정)

## 안건 템플릿
아래 템플릿을 채운 뒤 회의에 올린다.

- 안건명:
- 왜 만드는가:
- 해결할 플레이어 문제:
- EGO 헌법 연결 조항:
- 기대 효과:
- 리스크:
- 검증 방법:
- 실패 기준/중단 조건:
- 출시 후 관측 지표:
- 결론: 진행 / 보류 / 삭제
